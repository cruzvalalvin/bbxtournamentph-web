## 🔄 Keeping Documentation Updated

### When to Update Documentation

#### Architecture Changes
Update `architecture.md` when:
- Adding new layers or services
- Changing tech stack
- Modifying data flow
- Making architectural decisions

#### Feature Development
Update relevant docs when:
- Implementing new features
- Changing existing functionality
- Adding new APIs or endpoints
- Modifying business logic

#### Branding Changes
Update `branding.md` when:
- Changing colors or typography
- Adding new UI components
- Updating brand voice
- Creating new assets

#### Development Process
Update rules docs when:
- Establishing new patterns
- Changing coding standards
- Adding new tools or libraries
- Updating best practices
